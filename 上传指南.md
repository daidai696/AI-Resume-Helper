## 🔧 如果推送失败（身份验证错误）

### 方案1：使用Personal Access Token（推荐）

1. **生成Token**：
   - 访问：https://github.com/settings/tokens
   - 点击 "Generate new token" → "Generate new token (classic)"
   - 过期时间：选择 "No expiration" 或 90天
   - 权限范围：勾选 `repo`（完整仓库访问权限）
   - 滚动到底部，点击 "Generate token"
   - **复制Token**（只显示一次！）

2. **配置Token到Git**：
   ```bash
   git remote set-url origin https://你的token@github.com/daidai696/AI-Resume-Helper.git
   ```
   例如：
   ```bash
   git remote set-url origin https://ghp_xxxxxxxxxxxx@github.com/daidai696/AI-Resume-Helper.git
   ```

3. **重新推送**：
   ```bash
   git push origin main
   ```

### 方案2：使用SSH密钥

1. **生成SSH密钥**：
   ```bash
   ssh-keygen -t ed25519 -C "3165845253@qq.com"
   ```
   - 按回车使用默认路径
   - 设置密码或直接回车

2. **复制公钥**：
   ```bash
   cat ~/.ssh/id_ed25519.pub
   ```
   复制输出的全部内容

3. **添加到GitHub**：
   - 访问：https://github.com/settings/keys
   - 点击 "New SSH key"
   - 标题：随便填（如"My PC"）
   - 密钥类型：Authentication Key
   - 粘贴公钥内容
   - 点击 "Add SSH key"

4. **修改远程URL为SSH**：
   ```bash
   git remote set-url origin git@github.com:daidai696/AI-Resume-Helper.git
   ```

5. **重新推送**：
   ```bash
   git push origin main
   ```
